# 1c-copilot-mcp

MCP-сервер для интеграции ИИ-ассистентов (Claude Desktop, VS Code, Cursor и др.) со специализированной нейросетью **1С:Напарник** (`code.1c.ai`). Предоставляет 5 инструментов для работы с платформой **1С:Предприятие**: проверка кода, поиск знаний, консультации.

## Возможности

- 5 MCP-инструментов: проверка кода, поиск по ИТС/БСП/конфигурациям, объяснение синтаксиса, исследование кодовой базы, консультации
- Чтение кода из файлов с контролем доступа (`file_path` + диапазон строк)
- Автоматическая инъекция версии платформы в каждый запрос
- Управление сессиями: повторное использование, TTL, лимит активных сессий
- NFKC-нормализация ответов, нормализация маркеров кода (````1С (BSL)```` → ````bsl````)
- Логирование взаимодействий (промпт → мысли модели → ответ)

## Инструменты

| Инструмент | Назначение | file_path |
 |---|---|---|
| `check_1c_code` | Комплексная проверка: синтаксис, логика, производительность, архитектура, стиль, стандарты, лучшие практики, экспертный разбор | ✅ |
| `search_1c_knowledge` | Поиск по ИТС, платформе, типовым конфигурациям, БСП | — |
| `ask_1c_ai` | Консультация (general / consulting / expertise) | — |
| `explain_1c_syntax` | Объяснение синтаксиса и объектов метаданных | — |
| `explore_1c_codebase` | Исследование объектов, связей и зависимостей | — |

Параметры `code`/`file_path`/`start_line`/`end_line` — единые для всех инструментов, работающих с кодом. Приоритет: `code` > `file_path`.

## Переменные окружения

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ONEC_AI_TOKEN` | — | **Обязательный.** Токен API code.1c.ai |
| `ONEC_AI_PLATFORM_VERSION` | `8.3.25` | Версия платформы (добавляется в каждый запрос) |
| `ONEC_AI_TIMEOUT` | `120` | Таймаут запроса, сек |
| `ALLOWED_CODE_PATHS` | `.` | Разрешённые пути для чтения файлов (`;` для списка, `*` без ограничений) |
| `ONEC_AI_LOG_FIELDS` | — | Поля для логирования: `prompt`, `reasoning`, `answer` через пробел или запятую. Если не задана — логирование отключено |

### Настройка логирования

Логирование включается только если задана `ONEC_AI_LOG_FIELDS`. Примеры:

```bash
# Только промпт и ответ
ONEC_AI_LOG_FIELDS="prompt answer"

# Только мысли модели
ONEC_AI_LOG_FIELDS="reasoning"

# Все поля
ONEC_AI_LOG_FIELDS="prompt reasoning answer"
```

Лог пишется в файл `1c-copilot.log` рядом с `.exe`. Разделители: пробел или запятая.

## Режим чтения из файла (file_path)

Вместо того чтобы ИИ-ассистент копировал код 1С и передавал его в каждом запросе, MCP-сервер **сам читает код напрямую из файла** проекта. ИИ достаточно указать путь к файлу и диапазон строк — сервер извлечёт нужный фрагмент и подставит в запрос.

**Преимущества перед стандартной передачей кода через ИИ-модель:**

- **Экономия контекста.** Код не занимает место в окне контекста LLM — он передаётся напрямую сервером, минуя модель. Это критично для больших модулей, которые не влезают в лимит токенов.
- **Исключение ошибок копирования.** При передаче через модель код может искажаться (пропуски строк, потеря отступов, изменение спецсимволов). Чтение из файла гарантирует точное содержимое.
- **Контроль доступа.** Переменная `ALLOWED_CODE_PATHS` ограничивает, из каких директорий сервер может читать файлы.

Инструменты, поддерживающие file_path: `check_1c_code`.

## Для ИИ-ассистентов

Файл **[1c-copilot.md](1c-copilot.md)** содержит полный свод правил работы с сервером для ИИ-модели: описание 5 инструментов, алгоритм выбора, правило `file_path`, типовые сценарии и важные замечания.

**Файл необходимо включить в контекст LLM.** Без него модель не знает ключевых правил — например, что код нужно передавать через `file_path` а не `code`, и какой инструмент выбрать под задачу. Файл написан от первого лица для ИИ.

### Инструкция по установке (для ИИ-модели)

Если пользователь хочет подключить `1c-copilot` к проекту, выполни следующие шаги.

**Шаг 1 — спроси пользователя: собирать сервер из исходников или скачать готовый?**

Задай пользователю один вопрос:
> "1c-copilot MCP-сервер можно установить двумя способами:
> 1. **Собрать из исходников** — нужен установленный Go (компилятор)
> 2. **Скачать готовый** — из последнего релиза на GitHub
>
> Какой способ выбираете?"

Дождись ответа пользователя.

**Шаг 2 — действуй по ответу пользователя:**

**Вариант A: если пользователь выбрал «собрать из исходников»**
1. Проверь, установлен ли Go. Выполни команду: `go version`
2. Если команда `go version` вернула ошибку (Go не установлен):
   - Сообщи пользователю: «Go не найден. Предлагаю скачать готовый бинарник из последнего релиза.»
   - Если пользователь согласился — переходи к варианту B.
   - Если пользователь настаивает на сборке — сообщи, что нужно установить Go (https://go.dev/dl/) и повторить попытку.
3. Если Go установлен:
   - Клонируй репозиторий: `git clone https://github.com/EugeneVl/1c-copilot-mcp.git`
   - Перейди в папку: `cd 1c-copilot-mcp`
   - Собери: `go build`
   - Бинарник `1c-copilot-mcp.exe` появится в текущей папке.

**Вариант B: если пользователь выбрал «Скачать готовый» (или согласился после отсутствия Go)**
1. Создай папку `1c-copilot` в корне проекта пользователя.
2. Скачай `https://github.com/EugeneVl/1c-copilot-mcp/releases/download/v0.1.0/1c-copilot-mcp.exe` и сохрани его в папку `1c-copilot/`.
3. Скачай `https://github.com/EugeneVl/1c-copilot-mcp/releases/download/v0.1.0/1c-copilot.md` и сохрани его в `1c-copilot/`.

**Шаг 3 — получи токен**
Попроси пользователя зарегистрироваться на [code.1c.ai](https://code.1c.ai) и получить токен API.

**Шаг 4 — узнай, как в этой среде настраиваются MCP-серверы**

Не пиши конфигурацию сразу. Сначала выясни, какой MCP-клиент используется и в каком формате он ожидает настройки.

1. **Исследуй проект.** Проверь, есть ли в проекте файлы конфигурации MCP:
   - `.vscode/settings.json` или `.vscode/mcp.json` — VS Code
   - `.cursor/mcp.json` — Cursor
   - `claude_desktop_config.json` — Claude Desktop (обычно в `%APPDATA%\Claude\`)
   - `.windsurf/settings.json` — Windsurf
   - `mcp.json` или `.mcp.json` в корне проекта — универсальный MCP-формат
   - Если файл найден — открой его и изучи структуру: где там `mcpServers`, какой формат `command` (строка или массив), есть ли поле `instructions`.
2. **Если ни одного файла нет** — спроси пользователя: «Какой MCP-клиент вы используете? (VS Code, Cursor, Claude Desktop, Windsurf или другой?)»
3. **Изучи документацию клиента**, если нужно — убедись, что знаешь правильную структуру конфигурации для этого клиента.
4. **Определи корректный путь** к бинарнику (`<корень_проекта>/1c-copilot/1c-copilot-mcp.exe`) или воспользуйся подстановкой переменной `${workspaceFolder}` / `{{WORKSPACE_FOLDER}}` в зависимости от формата.

**Шаг 5 — добавь конфигурацию MCP-сервера**

Опираясь на изученную структуру, добавь блок `1c-copilot` в `mcpServers`. Пример для универсального формата:

```json
{
  "instructions": ["1c-copilot/1c-copilot.md"],
  "mcpServers": {
    "1c-copilot": {
      "command": "полный\\путь\\к\\1c-copilot-mcp.exe",
      "env": {
        "ONEC_AI_TOKEN": "ваш_токен",
        "ONEC_AI_PLATFORM_VERSION": "8.3.25",
        "ONEC_AI_TIMEOUT": "120",
        "ALLOWED_CODE_PATHS": ".",
        "ONEC_AI_LOG_FIELDS": ""
      }
    }
  }
}
```

Адаптируй формат под конкретный клиент:
- **VS Code (settings.json):** используй свой синтаксис `"mcp": { "servers": { ... } }`, поддерживает `${workspaceFolder}`.
- **VS Code (mcp.json):** чистая секция `mcpServers`, путь можно через `${workspaceFolder}`.
- **Cursor (.cursor/mcp.json):** секция `mcpServers`, поддерживает `{{WORKSPACE_FOLDER}}`.
- **Claude Desktop:** `mcpServers` в `claude_desktop_config.json`, путь должен быть абсолютным.

**Шаг 6 — добавь 1c-copilot.md в контекст проекта**
Файл `1c-copilot.md` в папке `1c-copilot/` содержит правила работы с MCP сервером. Добавь его в правила/инструкции проекта (обычно это поле `instructions` в той же конфигурации MCP).

**Шаг 7 — попроси пользователя перезапустить IDE чтобы настройки применились**
