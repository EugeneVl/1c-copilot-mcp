# 1c-copilot-mcp

MCP-сервер для интеграции ИИ-ассистентов (Claude Desktop, VS Code, Cursor и др.) со специализированной нейросетью **1С:Напарник** (`code.1c.ai`). Предоставляет 9 инструментов для работы с платформой **1С:Предприятие**: проверка и ревью кода, генерация и редактирование, поиск знаний, консультации.

## Возможности

- 9 MCP-инструментов: проверка ошибок, ревью стиля, генерация кода, редактирование, поиск по ИТС/БСП/конфигурациям, объяснение синтаксиса, документирование, исследование кодовой базы, консультации
- Чтение кода из файлов с контролем доступа (`file_path` + диапазон строк)
- Автоматическая инъекция версии платформы в каждый запрос
- Управление сессиями: повторное использование, TTL, лимит активных сессий
- NFKC-нормализация ответов, нормализация маркеров кода (````1С (BSL)```` → ````bsl````)
- Логирование взаимодействий (промпт → мысли модели → ответ)

## Быстрый старт

```bash
git clone https://github.com/EugeneVl/1c-copilot-mcp.git
cd 1c-copilot-mcp
go build
```

Укажите токен и подключите к MCP-клиенту:

```json
{
  "mcpServers": {
    "1c-copilot": {
      "command": "путь\\к\\1c-copilot-mcp.exe",
      "env": {
        "ONEC_AI_TOKEN": "ваш_токен"
      }
    }
  }
}
```

Токен получить на [code.1c.ai](https://code.1c.ai).

## Инструменты

| Инструмент | Назначение | file_path |
|---|---|---|
| `check_1c_code` | Комплексная проверка: синтаксис, логика, производительность, архитектура | ✅ |
| `review_1c_code` | Ревью стиля и стандартов (PascalCase, методика, DRY) | ✅ |
| `modify_1c_code` | Рефакторинг и редактирование по инструкции | ✅ |
| `generate_1c_code` | Генерация кода с нуля (full/module/procedure/function) | — |
| `search_1c_knowledge` | Поиск по ИТС, платформе, типовым конфигурациям, БСП | — |
| `ask_1c_ai` | Консультация (general / consulting / expertise) | — |
| `explain_1c_syntax` | Объяснение синтаксиса и объектов метаданных | — |
| `explore_1c_codebase` | Исследование объектов, связей и зависимостей | — |
| `generate_docstring` | Документирующий комментарий для процедуры/функции | ✅ |

Параметры `code`/`file_path`/`start_line`/`end_line` — единые для всех инструментов, работающих с кодом. Приоритет: `code` > `file_path`.

## Переменные окружения

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ONEC_AI_TOKEN` | — | **Обязательный.** Токен API code.1c.ai |
| `ONEC_AI_BASE_URL` | `https://code.1c.ai` | Базовый URL API |
| `ONEC_AI_PLATFORM_VERSION` | `8.3.25` | Версия платформы (добавляется в каждый запрос) |
| `ONEC_AI_UI_LANGUAGE` | `russian` | Язык интерфейса |
| `ONEC_AI_PROGRAMMING_LANGUAGE` | `1C (BSL)` | Язык программирования |
| `ONEC_AI_TIMEOUT` | `30` | Таймаут запроса, сек |
| `ALLOWED_CODE_PATHS` | `.` | Разрешённые пути для чтения файлов (`;` для списка, `*` без ограничений) |
| `MAX_ACTIVE_SESSIONS` | `10` | Максимум одновременных сессий |
| `SESSION_TTL` | `3600` | Время жизни сессии, сек |
| `ONEC_AI_LOG_FIELDS` | — | Поля для логирования: `prompt`, `reasoning`, `answer` через пробел или запятую. Если не задана — логирование отключено |

### Настройка логирования

Логирование включается только если задана `ONEC_AI_LOG_FIELDS`. Примеры:

```bash
# Только промпт и ответ
ONEC_AI_LOG_FIELDS="prompt answer"

# Только мысли модели
ONEC_AI_LOG_FIELDS="reasoning"

# Все поля
ONEC_AI_LOG_FIELDS="prompt reasoning answer"
```

Лог пишется в файл `1c-copilot.log` рядом с `.exe`. Разделители: пробел или запятая.

## Режим чтения из файла (file_path)

Вместо того чтобы ИИ-ассистент копировал код 1С и передавал его в каждом запросе, MCP-сервер **сам читает код напрямую из файла** проекта. ИИ достаточно указать путь к файлу и диапазон строк — сервер извлечёт нужный фрагмент и подставит в запрос.

**Преимущества перед стандартной передачей кода через ИИ-модель:**

- **Экономия контекста.** Код не занимает место в окне контекста LLM — он передаётся напрямую сервером, минуя модель. Это критично для больших модулей, которые не влезают в лимит токенов.
- **Исключение ошибок копирования.** При передаче через модель код может искажаться (пропуски строк, потеря отступов, изменение спецсимволов). Чтение из файла гарантирует точное содержимое.
- **Целостность.** Даже если код большой, он попадёт в запрос без искажений и обрезаний, которые могут возникнуть при копировании через буфер обмена или экран.
- **Контроль доступа.** Переменная `ALLOWED_CODE_PATHS` ограничивает, из каких директорий сервер может читать файлы.

Инструменты, поддерживающие file_path: `check_1c_code`, `review_1c_code`, `modify_1c_code`, `generate_docstring`.

## Для ИИ-ассистентов

Файл **[1c-copilot.md](1c-copilot.md)** содержит полный свод правил работы с сервером для ИИ-модели: описание 9 инструментов, алгоритм выбора, правило `file_path`, типовые сценарии и важные замечания.

**Файл необходимо включить в контекст LLM.** Без него модель не знает ключевых правил — например, что код нужно передавать через `file_path` а не `code`, и какой инструмент выбрать под задачу. Файл написан от первого лица для ИИ.

**Как включить в контекст:**
- **Kilo:** добавить содержимое в `.kilo/agent/` или project rules
- **Cursor:** добавить в `.cursor/rules/`
- **Claude Code:** добавить в `.claude/` или project instructions
- **VS Code (Cline/Continue):** добавить в custom instructions проекта

### Инструкция по установке (для ИИ-модели)

Если пользователь хочет подключить `1c-copilot` к проекту, покажи ему этот блок:

---

**1. Скачай сервер.** Собери из исходников:
```bash
git clone https://github.com/EugeneVl/1c-copilot-mcp.git
cd 1c-copilot-mcp
go build
```
Готовый `.exe` — в [релизах](https://github.com/EugeneVl/1c-copilot-mcp/releases).

**2. Получи токен** на [code.1c.ai](https://code.1c.ai).

**3. Добавь конфигурацию** в настройки MCP-клиента:
```json
{
  "mcpServers": {
    "1c-copilot": {
      "command": "путь\\к\\1c-copilot-mcp.exe",
      "env": {
        "ONEC_AI_TOKEN": "ваш_токен"
      }
    }
  }
}
```

**Где находятся файлы настроек:**
| Клиент | Путь |
|---|---|
| Claude Desktop | `%APPDATA%\Claude\claude_desktop_config.json` |
| Kilo | `kilo.json` → поле `mcpServers` |
| Cursor | `.cursor/mcp.json` |
| VS Code (Cline) | Cline → Настройки → MCP Servers |
| Windsurf | `~/.codeium/windsurf/mcp_config.json` |

**4. Перезапусти MCP-клиент** и включи `1c-copilot.md` в контекст проекта.
